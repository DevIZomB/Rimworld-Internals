Rand & RNG

Determinismus-Fundament von RimWorld — 12 Kern-Mechanismen

Warnung
Jeder Rand-Aufruf in einem Mod verschiebt den globalen iterations-Counter. Für nicht-gameplay Zufall IMMER PushState/PopState verwenden!
#1 Globaler State Verse/Rand.cs:13-17

MP-Kritisch — Zwei Felder definieren den gesamten RNG-State: seed (uint) und iterations (uint). Jeder Value-Aufruf erhöht iterations um 1. Determinismus hängt davon ab, dass alle Clients exakt gleich viele Rand-Calls machen.

#2 MurmurHash Verse/MurmurHash.cs:1-34

Pure Hash-Funktion: GetInt(seed, iterations) → deterministischer int. Keine Seiteneffekte, kein State. Wird von Rand.Value intern verwendet.

#3 Initialisierung & Seed Verse/Rand.cs

Seed wird beim Spielstart gesetzt. iterations beginnt bei 0 und zählt monoton hoch. Nach einem Save/Load ist TicksGame identisch, aber der Rand-State wird nicht gespeichert — er ergibt sich aus der exakten Replay-Sequenz.

Verbindung: → RI-02.08 (TickManager ExposeData)

#4 State Stack — PushState/PopState Verse/Rand.cs:440-454

Stack-basiertes State-Management: PushState() speichert aktuellen seed+iterations auf Stack und setzt neuen Seed. PopState() stellt alten State wieder her. MUSS für UI/nicht-gameplay Rand verwendet werden.

Eigene Mods: immer PushState/PopState um visuellen Zufall (Partikel, UI) vom Gameplay-Rand zu isolieren.

#5 RandBlock (using-Pattern) Verse/Rand.cs

Convenience-Wrapper: using(new RandBlock(seed)) { ... } ruft automatisch PushState/PopState auf. Sicherer als manuelles Push/Pop da Exceptions den State nicht korrumpieren.

#6 Value & Basis-Operationen Verse/Rand.cs:35-51

Rand.Value → float [0,1). Rand.Int, Rand.Bool, Rand.Range(min,max). Jeder Aufruf erhöht iterations. Rand.Chance(f) = Value < f.

#7 Seeded Varianten Verse/Rand.cs

Rand.ValueSeeded(seed) — erzeugt einen deterministischen Wert basierend auf einem expliziten Seed, ohne den globalen State zu verändern. Wird z.B. von Need_Food verwendet.

MP-Kritisch — Obwohl stateless, muss der Input-Seed auf allen Clients identisch sein.

#8 MTBEventOccurs Verse/Rand.cs:509-554

MP-Kritisch — Mean Time Between Events: Berechnet Wahrscheinlichkeit pro Tick basierend auf MTB-Tagen. Verbraucht einen Rand.Value-Call. Wird massiv genutzt (Krankheiten, Mental Breaks, Events).

#9 RandSeedForHour Verse/Rand.cs

Thing-Extension: Generiert einen Seed basierend auf Thing.thingIDNumber + GenTicks.TicksAbs / 2500. Gibt pro Stunde Spielzeit einen stabilen Seed — wird für stundenweise Variation genutzt (z.B. Wanderer-Richtung).

Verbindung: → RI-02.09 (TicksAbs)

#10 ValueAsync (Stateless) Verse/Rand.cs

Einzelwert-Berechnung ohne globalen State: MurmurHash.GetInt(seed, 0). Für Fälle wo ein einzelner deterministischer Wert gebraucht wird ohne den Rand-Counter zu belasten.

#11 Gaussian, ByCurve, InsideUnitCircle Verse/Rand.cs:137-507

Erweiterte Zufallsfunktionen: Gaussian (Box-Muller, 2 Rand-Calls), ByCurve (gewichtete Kurve), InsideUnitCircle/InsideUnitCircleVec3 (Rejection Sampling, variable Rand-Calls).

MP-Kritisch — InsideUnitCircle verbraucht variable Anzahl Rand-Calls je nach Rejection — besonders desync-gefährlich.

#12 Range, Chance, Element Utilities Verse/Rand.cs

Rand.Range(min, max), Rand.Chance(float), list.RandomElement(), list.RandomElementByWeight(). Alle verbrauchen mindestens 1 Rand-Call. TryRandomElement verbraucht 0 Calls wenn Liste leer ist.
