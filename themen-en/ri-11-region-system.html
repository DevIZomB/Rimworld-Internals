RI-11 Namespace: Verse

Region System

12×12 pathfinding building blocks & danger detection

#1 Region Verse/Region.cs:1-610

12×12 building block of the pathfinding system. Contains closedIndex[8] for parallel BFS traversals, ListerThings for fast thing queries, and danger cache for pawn AI.

#2 RegionGrid Verse/RegionGrid.cs:1-253

Region[] array across the entire map. Auto-rebuild on terrain/building changes. UpdateClean processes max 16 dirty regions per frame.

#3 RegionTraverser (BFS) Verse/RegionTraverser.cs:1-242

BFS traversal with worker pool (8 workers). Uses closedIndex trick: instead of a visited HashSet, a global counter per worker is incremented — O(1) instead of O(n) reset.

Use BreadthFirstTraverse for custom reachability queries.

#4 District Verse/District.cs:1-338

Groups multiple regions into a logical area. Room membership, OpenRoofCountStopAt with lazy evaluation for performance.

#5 RegionType (Flags Enum) Verse/RegionType.cs:1-16

Four types: Normal (walkable), Portal (doors), Fence (fences, blocks animals), Impassable (walls/rocks).

#6 DangerFor (Danger Detection) Verse/Region.cs

Region.DangerFor(Pawn) checks room temperature against the pawn's SafeTemperatureRange. Result is cached. Pawns automatically avoid regions outside their temperature range.

Danger levels: None, Some (hostile presence), Deadly (temperature/vacuum).

Connection: → Temperature System (v2 main page), → RI-06 (Pawn ComfortableTemperatureRange)

#7 Region → District → Room Hierarchy

Region (12×12 cells) → District (connected regions of same type) → Room (districts connected via portal regions/doors). A room can have multiple districts. Room.UsesOutdoorTemperature determines whether the room counts as outdoor (≥25% open OR TouchesMapEdge).

Connection: → Temperature System (Room Hierarchy section)
