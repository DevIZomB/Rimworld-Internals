RI-04 Namespace: Verse

Thing Hierarchy

Base class of all game objects — 10 core mechanisms

#1 Thing Core Fields & State Machine Verse/Thing.cs:17-69

Core fields of every thing: thingIDNumber, def, mapIndexOrState (state machine: -3=Discarded, -2=Destroyed, -1=Unspawned, ≥0=Map-Index).

#2 DoTick Dispatch Verse/Thing.cs

TickList calls thing.DoTick(), which dispatches to Tick(), TickRare() or TickLong() based on def.tickerType. Normal = every tick, Rare = every 250, Long = every 2000.

Connection: → RI-02.03 (TickList Bucket System)

#3 SpawnSetup Verse/Thing.cs:803-928

15+ subsystem registrations on spawn: ThingGrid, RegionDirtyer, ListerThings, CoverGrid, edificeGrid, GlowGrid, MapMeshDirty, gasGrid, mapTemperature, TickManager, etc. Order is fixed.

MP-Critical — SpawnSetup triggers RegionDirtyer → room rebuild → equalizeCells.Shuffle() → Rand call.

#4 DeSpawn Verse/Thing.cs:945-1036

Mirror deregistration of SpawnSetup: All subsystems are notified in reverse order. mapIndexOrState is reset.

#5 Destroy & Kill Verse/Thing.cs

Destroy(DestroyMode) calls DeSpawn and sets state to Destroyed (-2). Kill(DamageInfo) is the gameplay-side variant that triggers damage events and may spawn corpse/wreckage.

DestroyModes: Vanish, WillReplace, Deconstruct, FailConstruction, Refund, KillFinalize, KillFinalizeLeavingsOnly.

#6 ExposeData (Serialization) Verse/Thing.cs

Save/load system: Saves def, ID, position, rotation, faction, hitPoints, stuff. PostExposeData() hook for subclasses. Comps serialize themselves via CompExposeData().

#7 PostMake & PostPostMake Verse/Thing.cs

Factory hooks on creation: PostMake() is called after ThingMaker.MakeThing, PostPostMake() after that. Comps receive CompPostMake(). Ideal for initialization that shouldn't happen in the constructor.

#8 ThingWithComps Component System Verse/ThingWithComps.cs:1-221

THE modding mechanism: ThingComps are defined via XML and instantiated at runtime. GetComp<T> is optimized: direct is-checks for <3 comps, dictionary for ≥3 comps.

Custom ThingComps for new functionality via XML + C#.

#9 ThingWithComps Lifecycle Delegation Verse/ThingWithComps.cs

All lifecycle methods iterate over comps: PostSpawnSetup, PostDeSpawn, PostDestroy, CompTick, CompTickRare, CompTickLong. Comps automatically receive all events.

#10 Building Verse/Building.cs:1-684

Inherits from ThingWithComps. MinTickIntervalRate = 6 (building ticks at most every 6 ticks). Manages edificeGrid registration, paint system for colors, and roof collapse notification on destruction.

Connection: Building spawn/despawn → RegionDirtyer → room rebuild (→ RI-11)
