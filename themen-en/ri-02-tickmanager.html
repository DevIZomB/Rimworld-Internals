TickManager & Game Loop

Heartbeat of RimWorld — 9 core mechanisms

#1 TickManagerUpdate (Frame Loop) Verse/TickManager.cs:318-355

Called every Unity frame. Calculates from Time.deltaTime and TickRateMultiplier how many ticks to execute this frame. Has frametime jitter correction (line 326-333): when deltaTime ≈ curTimePerTick (±10%), exact value is used instead of deltaTime. Maximum wall-clock per frame: ~45ms (= 22 FPS minimum).

#3 TickList Bucket System Verse/TickList.cs:1-127

Distributes things into buckets based on GetHashCode() % TickInterval. Per Tick() only the bucket TicksGame % TickInterval is processed. Deferred register/deregister: new things enter the list only on the next Tick() call (avoids ConcurrentModification).

TickerTypeIntervalGame Time
Normal1 (every tick)60/s at speed 1
Rare250~4.2 seconds
Long2000~33 seconds

Things tick based on def.tickerType. IThingHolder is always ticked as Normal.

#4 TickRateMultiplier (Speed Logic) Verse/TickManager.cs:74-114

Superfast is dynamic: 18× without maps, 12× when NothingHappening (everyone asleep, no danger), 6× otherwise. Ultrafast: 150× without maps, 15× with maps.

Harmony postfix on TickRateMultiplier to add custom speed logic.

#5 TimeSlower (Event Brake) Verse/TimeSlower.cs:1-31

Forces normal speed after important events. SignalForceNormalSpeed() = 800 ticks (~13s), SignalForceNormalSpeedShort() = 240 ticks (~4s). Called on raids, hostile map generations, etc.

Call slower.SignalForceNormalSpeed() when your mod triggers an important event.

#6 NothingHappeningInGame Verse/TickManager.cs:245-282

Checks if "nothing is happening" on all maps — all human colonists are sleeping and no danger is active. Result is cached per tick. Determines whether superfast is 6× or 12×.

Patch NothingHappeningInGame to add custom "something is happening" logic.

#7 ExposeData (Serialization) Verse/TickManager.cs:284-290

Saves: ticksGameInt, gameStartAbsTick, startingYear, lastSettleTicks. TickLists are NOT saved — things re-register themselves on load.

Connection: Rand state is NOT saved (→ RI-01.03)

#8 TicksAbs vs TicksGame Verse/TickManager.cs:53-66

TicksGame = ticks since game start (begins at 0). TicksAbs = TicksGame + gameStartAbsTick (absolute time since year 0). gameStartAbsTick is set by the scenario and remains constant.

For time-based calculations use GenTicks.TicksAbs, not TicksGame.

Connection: RandSeedForHour uses TicksAbs (→ RI-01.09)

#9 TickerType & TimeSpeed (Enums) Verse/TickerType.cs, Verse/TimeSpeed.cs

TickerType: Never (0), Normal (1), Rare (250), Long (2000).
TimeSpeed: Paused (0×), Normal (1×), Fast (3×), Superfast (6-18×), Ultrafast (15-150×).

TickerType is set in ThingDef. TimeSpeed can be changed at runtime.
