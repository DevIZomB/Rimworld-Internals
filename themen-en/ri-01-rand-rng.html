Rand & RNG

Warning
Every Rand call in a mod shifts the global iterations counter. For non-gameplay randomness ALWAYS use PushState/PopState!
#1 Global State Verse/Rand.cs:13-17

MP-Critical — Two fields define the entire RNG state: seed (uint) and iterations (uint). Every Value call increments iterations by 1. Determinism depends on all clients making exactly the same number of Rand calls.

#2 MurmurHash Verse/MurmurHash.cs:1-34

Pure hash function: GetInt(seed, iterations) → deterministic int. No side effects, no state. Used internally by Rand.Value.

#3 State Stack PushState/PopState Verse/Rand.cs:440-454

Stack-based state management: PushState() saves current seed+iterations to stack and sets new seed. PopState() restores previous state. MUST be used for UI/non-gameplay Rand.

#4 Value & Base Operations Verse/Rand.cs:35-51

Rand.Value → float [0,1). Rand.Int, Rand.Bool, Rand.Range(min,max). Each call increments iterations. Rand.Chance(f) = Value < f.

#5 MTBEventOccurs Verse/Rand.cs:509-554

MP-Critical — Mean Time Between Events: Calculates probability per tick based on MTB days. Consumes one Rand.Value call. Heavily used (diseases, mental breaks, events).

#6 Gaussian, ByCurve, InsideUnitCircle Verse/Rand.cs:137-507

Advanced random functions: Gaussian (Box-Muller, 2 Rand calls), ByCurve (weighted curve), InsideUnitCircle/InsideUnitCircleVec3 (rejection sampling, variable Rand calls).

InsideUnitCircle consumes variable number of Rand calls depending on rejection — especially desync-dangerous.
