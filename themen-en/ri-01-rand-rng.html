RI-01 Namespace: Verse

Rand & RNG

Determinism foundation of RimWorld — 12 core mechanisms

Warning
Every Rand call in a mod shifts the global iterations counter. For non-gameplay randomness ALWAYS use PushState/PopState!
#1 Global State Verse/Rand.cs:13-17

MP-Critical — Two fields define the entire RNG state: seed (uint) and iterations (uint). Every Value call increments iterations by 1. Determinism depends on all clients making exactly the same number of Rand calls.

#2 MurmurHash Verse/MurmurHash.cs:1-34

Pure hash function: GetInt(seed, iterations) → deterministic int. No side effects, no state. Used internally by Rand.Value.

#3 Initialization & Seed Verse/Rand.cs

Seed is set at game start. iterations starts at 0 and counts up monotonically. After a save/load, TicksGame is identical, but the Rand state is not saved — it results from the exact replay sequence.

Connection: → RI-02.08 (TickManager ExposeData)

#4 State Stack — PushState/PopState Verse/Rand.cs:440-454

Stack-based state management: PushState() saves current seed+iterations to stack and sets new seed. PopState() restores old state. MUST be used for UI/non-gameplay Rand.

Custom mods: always use PushState/PopState to isolate visual randomness (particles, UI) from gameplay Rand.

#5 RandBlock (using-Pattern) Verse/Rand.cs

Convenience wrapper: using(new RandBlock(seed)) { ... } automatically calls PushState/PopState. Safer than manual Push/Pop since exceptions won't corrupt the state.

#6 Value & Basic Operations Verse/Rand.cs:35-51

Rand.Value → float [0,1). Rand.Int, Rand.Bool, Rand.Range(min,max). Every call increments iterations. Rand.Chance(f) = Value < f.

#7 Seeded Variants Verse/Rand.cs

Rand.ValueSeeded(seed) — produces a deterministic value based on an explicit seed, without modifying the global state. Used e.g. by Need_Food.

MP-Critical — Although stateless, the input seed must be identical on all clients.

#8 MTBEventOccurs Verse/Rand.cs:509-554

MP-Critical — Mean Time Between Events: Calculates probability per tick based on MTB days. Consumes one Rand.Value call. Used extensively (diseases, mental breaks, events).

#9 RandSeedForHour Verse/Rand.cs

Thing extension: Generates a seed based on Thing.thingIDNumber + GenTicks.TicksAbs / 2500. Gives a stable seed per game hour — used for hourly variation (e.g. wanderer direction).

Connection: → RI-02.09 (TicksAbs)

#10 ValueAsync (Stateless) Verse/Rand.cs

Single-value calculation without global state: MurmurHash.GetInt(seed, 0). For cases where a single deterministic value is needed without burdening the Rand counter.

#11 Gaussian, ByCurve, InsideUnitCircle Verse/Rand.cs:137-507

Advanced random functions: Gaussian (Box-Muller, 2 Rand calls), ByCurve (weighted curve), InsideUnitCircle/InsideUnitCircleVec3 (rejection sampling, variable Rand calls).

MP-Critical — InsideUnitCircle consumes variable number of Rand calls depending on rejection — especially desync-dangerous.

#12 Range, Chance, Element Utilities Verse/Rand.cs

Rand.Range(min, max), Rand.Chance(float), list.RandomElement(), list.RandomElementByWeight(). All consume at least 1 Rand call. TryRandomElement consumes 0 calls when list is empty.
