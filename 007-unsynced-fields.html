Potential Sync Gaps

Fields where UI writes simulation state and the MP mod does not appear to sync the write path. Sorted by gameplay relevance.

User-Facing (Could Affect Normal Gameplay)

MEDIUM Designator_Plan_Add.colorDef

File: RimWorld/Designator_Plan_Add.cs

What happens: Player selects plan color in ProcessInput() (UI) → color is stored → DesignateSingleCell() (SIM) uses it to create designations. If two players use the plan designator simultaneously with different colors, the color state could differ.

Impact: Different plan colors per client. Not a simulation desync (plans don't affect gameplay), but visible inconsistency.

MEDIUM CompPlantable.plantCells — Cancel path

File: RimWorld/CompPlantable.cs

What happens: BeginTargeting (adding cells) IS synced via SyncDelegate. But the "Cancel All" gizmo in CompGetGizmosExtra() clears the list — this clear operation may not be synced.

Impact: If one player cancels planting via gizmo, other clients still have cells queued → pawns get different job targets.

Issues found through AI code analysis that go beyond simple field access patterns. These are systemic risks, not individual fields.
